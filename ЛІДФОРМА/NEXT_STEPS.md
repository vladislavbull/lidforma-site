# Наступні кроки ЛІДФОРМИ

Оновлено: 17 липня 2026 року.

## Головний фокус

Отримати перші якісні звернення, не руйнуючи навчання рекламної кампанії, паралельно підготувати докази компетентності та надійну систему збереження проєкту.

## 1. Meta Ads — найближчі 72 години

- [ ] Перевірити актуальний статус усіх трьох оголошень і дату початку показів.
- [ ] Не змінювати бюджет, аудиторію, optimization event або креативи протягом перших 48–72 годин без критичної причини.
- [ ] Через 24 години зафіксувати spend, impressions, reach, CPM, link clicks, CTR link, CPC link, landing page views, Leads і CPL по кожному оголошенню.
- [ ] Через 72 години порівняти A/B/C і визначити, де проблема: покази, кліки, завантаження сторінки, форма або якість ліда.
- [ ] Перевірити, що реальні Lead-події з реклами відображаються у Events Manager без дублювання.
- [ ] Не вимикати оголошення лише через нерівномірний розподіл перших кількох гривень.

- [ ] Налаштувати нормальний цикл: локальна зміна → перевірка → commit → push → Vercel deployment.

Поточний технічний блокер: у середовищі немає GitHub CLI `gh`, а прямий `git clone` заблоковано мережевим доступом shell. Не створювати окрему локальну історію через `git init` і не робити force push. Наступний безпечний варіант — встановити й авторизувати `gh`, після чого клонувати `main` в окрему папку, перенести перевірені локальні доповнення та відкрити draft PR.
