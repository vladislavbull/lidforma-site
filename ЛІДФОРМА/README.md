# ЛІДФОРМА — памʼять проєкту

Ця папка є постійним контекстом проєкту ЛІДФОРМА для Codex та наступних робочих діалогів.

## З чого почати новий діалог

1. Відкрити робочий простір:
   `/Users/vladislavmoskalkov/Documents/Codex/2026-07-10/new-chat-2`
2. Попросити Codex прочитати всі Markdown-файли з папки `ЛІДФОРМА`.
3. Перевірити фактичний стан файлів, сайту, Git та зовнішніх сервісів перед змінами.
4. Продовжити з першого актуального пункту в `NEXT_STEPS.md`.

Готовий стартовий запит:

> Продовжуємо роботу над ЛІДФОРМОЮ. Спочатку повністю прочитай ЛІДФОРМА/README.md, AGENTS.md, PROJECT_CONTEXT.md, DECISIONS.md і NEXT_STEPS.md. Потім перевір фактичний стан робочої папки та продовжуй з актуального наступного кроку. Не повторюй уже завершену роботу і не змінюй активну рекламу чи продакшен без мого підтвердження.

## Документи

- `AGENTS.md` — постійні правила роботи над проєктом.
- `PROJECT_CONTEXT.md` — бізнес, бренд, послуги, технічна схема та важливі посилання.
- `DECISIONS.md` — прийняті рішення та причини.
- `NEXT_STEPS.md` — поточний стан, пріоритети та контрольні списки.

## Де знаходяться робочі файли

Сайт і матеріали зараз знаходяться на один рівень вище, у корені робочої папки. Це зроблено навмисно, щоб не зламати чинний звʼязок із Vercel.

Основні файли:

- `../index.html`, `../styles.css`, `../script.js` — головна сторінка.
- `../brief.html`, `../brief.css`, `../brief-logo.css`, `../brief.js` — бриф.
- `../apps-script/Code.gs` — Google Apps Script для заявок, таблиць, Telegram та email.
- `../meta-ad-creatives/` — рекламні креативи Meta.
- `../THREADS_100_ЛІДФОРМА.md` — підготовлений контент Threads.
- `../logo-*` — активи логотипа.
- `../.vercel/project.json` — локальний звʼязок із проєктом Vercel.

## Правило актуальності

Після важливого рішення або завершення етапу потрібно:

1. додати рішення в `DECISIONS.md`;
2. оновити статус у `NEXT_STEPS.md`;
3. за потреби актуалізувати `PROJECT_CONTEXT.md`;
4. зробити Git-коміт, коли локальну папку буде коректно підключено до репозиторію.

## Безпека

У цій папці не можна зберігати паролі, токени Telegram, SMTP-паролі, cookies, API-ключі або персональні дані клієнтів. Секрети мають зберігатися у властивостях Apps Script, змінних середовища Vercel або іншому захищеному сховищі.
